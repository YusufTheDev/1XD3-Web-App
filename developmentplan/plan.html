Infinite Travel - Development Plan

1. Names

  • Team: DreamWeaver
  • Names: Tianyan, Yusuf, David, Edward
  • Client Name: Darim Khan
  • Application Name: Infinite Travel

2. Purpose

The project is an endless runner game where users can log in, play, earn a score, and appear on a leaderboard. Players can upgrade abilities like double jumps, extra lives, and weapons to enhance gameplay. The game features a clean, minimalist interface and is accessible on both mobile and desktop.

3. Data Description

User Information Table

Why is this needed? Stores user login details and their highest score.

  • Fields: user_id (primary key), username, password_hash, email, best_score

Score Leaderboard Table

Why is this needed? Tracks user scores for leaderboard ranking.

  • Fields: score_id (primary key), user_id, score

4. Functions Performed by the Application

User Authentication

  • Login: Users log in with their credentials (Database: SELECT)
  • Register: Users create an account (Database: INSERT)
  • Logout: Users log out (Client-side: Clear session/cookies)

Playing the Game

  • Start Game: Loads the game and initializes player stats.
  • Pause/Resume Game: Players can pause/resume the game.
  • Game Over: Ends the session and displays stats.

Score & Leaderboard Management

  • Save Score: Saves the player's score to the leaderboard (Database: INSERT)
  • Retrieve Leaderboard: Displays the top scores (Database: SELECT)

Upgrades & Shop System

  • Purchase Upgrades: Players buy power-ups (Database: UPDATE)
  • Equip/Unequip Upgrades: Allows players to activate/deactivate abilities.

Profile Management

  • View Profile: Displays user stats and scores (Database: SELECT)
  • Change Password: Updates user password (Database: UPDATE)

5. The Role of PHP and JavaScript

PHP: Handles user login, registration, saving scores, and retrieving leaderboard data.

JavaScript: Used for game logic inside the Canvas element, including rendering, player movement, collision detection, and UI interactions.

6. Roles of Team Members

  • Tianyan: Handles login features and leaderboard management.
  • Yusuf: Manages core game functionality and score tracking.
  • David: Designs character, animations, and game environment.
  • Edward: Implements attack mechanics and interactions.

7. The Increments

Target Date: April 3rd, 2025

Functionality for Each Team Member

  • Tianyan: User login system (PHP backend, JavaScript for AJAX)
  • Yusuf: Game movement logic and progress handling (JavaScript)
  • David: Basic character design and Canvas setup (JavaScript, CSS)
  • Edward: Scrolling background animation (JavaScript)

Description of the Increment

This first increment will focus on foundational features: the user login system, character movement, and game environment setup. Players will be able to log in and start the game, with a simple character and a scrolling background representing the endless runner environment.

8. Client Feedback

Feedback: The client expressed approval of the idea and had no major concerns. The only requested change was to add some form of attacking to make the game more engaging.

Client Quote: "It would be more fun if players could attack obstacles instead of just avoiding them."

Implementation Plan: Introduce objects that move across the screen and can be destroyed by the player using ranged attacks, rewarding them with extra points or other benefits.
